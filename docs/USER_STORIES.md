# Uni-Bridge: User Stories & Requirements

This document outlines the core requirements for the Uni-Bridge platform, managed by the Product Owner (Smarika Singh Thakuri).

## 1. Skill-Based Search (Core Feature)
**User Story:** As a student struggling with a specific module, I want to search for peers by module code (e.g., CS4001), so that I can find a study buddy with the right expertise.
* **Acceptance Criteria:*  Search bar accepts alphanumeric module codes.
    * Results display student names and their "Expert" modules.

## 2. Student Profile & Skill-Swap
**User Story:** As a student proficient in a subject, I want to list my "Expert" skills and "Learning" needs, so that I can exchange my knowledge for help in other areas.
* **Acceptance Criteria:*  Users can edit a profile page.
    * Skills are tagged as 'Offering' or 'Requesting'.

## 3. Safety & Meeting Logistics
**User Story:** As a new user, I want to see a list of verified safe meeting zones on campus, so that I feel comfortable meeting a peer for the first time.
* **Acceptance Criteria:*  Links to campus library hours and Student Union locations.
    * Safety disclaimer must be accepted during registration.

## 4. Academic Integrity
**User Story:** As a member of the university community, I want the platform to have a strict policy against cheating, so that the value of our degrees is protected.
* **Acceptance Criteria:* Users must sign an "Anti-Collusion Agreement" before messaging.
